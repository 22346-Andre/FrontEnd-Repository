## ❓ Perguntas Esperadas (e Respostas)

### "Como funciona a integração com o backend?"
> *"O sistema já está mapeado para todos os endpoints necessários. Temos DTOs, interfaces e chamadas preparadas. É só conectar o Spring Boot. Todos os endpoints estão documentados no arquivo SISTEMA-COMPLETO.md"*

### "Comandos de voz funcionam em todos os navegadores?"
> *"A Web Speech API é nativa do Chrome, Edge e Safari, que representam mais de 85% do mercado. Para outros navegadores, o sistema continua funcionando normalmente sem essa feature."*

### "E se o usuário não quiser usar comandos de voz?"
> *"É totalmente opcional! Todas as funcionalidades têm alternativas clássicas (botões, formulários). Comandos de voz são um diferencial, não uma dependência."*

### "O sistema está pronto para produção?"
> *"O frontend está 100% funcional com dados mock. Para produção, precisamos apenas conectar ao backend Java e configurar um banco de dados real. A arquitetura já está preparada."*

### "Quanto custaria implementar comandos de voz?"
> *"Zero! Web Speech API é nativa do navegador, sem custos de APIs externas como Google Cloud Speech ou AWS Transcribe. É um diferencial competitivo sem custo adicional."*
